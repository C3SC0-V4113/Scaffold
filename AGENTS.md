# AGENTS.md

This repository is a standalone Node/TypeScript ESM CLI package. It generates new Next.js apps with shadcn, quality tooling, docs, hooks, and optional tests.

## Quality Gate

Run this before claiming implementation complete:

```bash
cmd /c npm run check
```

`npm run check` performs typechecking, test execution, and build. If it cannot run, report the exact command and error.

CI mirrors this: `.github/workflows/ci.yml` runs `npm run check` plus the quick E2E tier (dry-run scenarios) on ubuntu and windows for every push/PR. `.github/workflows/e2e.yml` runs the full scenario matrix (one parallel job per real scenario, generated from `scripts/cli-e2e.mjs --list` — never hardcode scenario names in workflows) on PRs that touch `src/` or `scripts/`, nightly against upstream `@latest` tools, and on manual dispatch. A failed nightly run opens a GitHub issue.

## Development Rules

- Source code lives in `src/`.
- Tests live in `tests/` and should cover functional changes.
- `dist/` is generated by `npm run build` (tsup, single bundled `dist/index.js`); do not edit it manually.
- Package behavior should remain testable through the `Executor` abstraction so dry-runs do not execute real commands or write files.
- Preserve support for `npm`, `pnpm`, and `bun` unless a change explicitly narrows scope.
- Keep generated project docs generic; do not hardcode Cost Console or other project-specific language.
- `src/cli-metadata.ts` is the single source of truth for CLI options and scenarios. When you add or change a flag, update it there — `--help`, `info --json`, `llms.txt`, the README table, and `skills/purrfold/SKILL.md` all mirror it.
- `src/versions.json` is the single source of truth for the dependency versions pinned into generated apps. `src/installers/config-model.ts`, the E2E harness/scenarios, and the tests all read from it — never hardcode a generated-app version anywhere else. Renovate bumps these pins automatically (custom regex manager in `renovate.json`); manual bumps must run the CLI E2E suite (`npm run test:e2e:cli`). Pin bumps change generated-app behavior, so they need a changeset before release.

## Releases

Versioning and `CHANGELOG.md` are managed with Changesets. For any user-facing change:

1. `npm run changeset` — describe the change and pick the bump type.
2. `npm run changeset:version` — apply bumps and update `CHANGELOG.md`; commit.
3. `npm run release` — runs `npm run check` (via `prepublishOnly`), publishes, and tags.

Do not hand-edit `CHANGELOG.md` or bump `version` manually.

## Agent-facing docs

purrfold is meant to be driven by agents. `llms.txt` (shipped in the package) and
`skills/purrfold/SKILL.md` (Claude Code skill) document the install command and an
intent→command table. Keep both in sync with `src/cli-metadata.ts`.
